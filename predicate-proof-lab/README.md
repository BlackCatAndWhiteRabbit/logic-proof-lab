# 谓词逻辑证明实验室

这是一个面向一阶谓词逻辑 Hilbert 公理证明训练的本地网页应用。它沿用命题逻辑证明实验室的页面结构，支持目标定理、证明步骤检查、MP、UG、定理库保存、定理依赖图和演绎子证明。

## 打开方式

方式一：直接用浏览器打开 `index.html`。

方式二：双击 `run-site.bat`，启动本地服务后访问 `http://127.0.0.1:5600`。

如果系统里没有安装 Node.js，`run-site.bat` 会优先尝试使用 Codex 自带的 Node.js；仍然找不到时，会直接打开 `index.html`。

方式三：如果系统已经安装 Node.js 和 npm，也可以运行：

```bash
npm start
```

## 支持语法

- `~A`：否定
- `A -> B`：蕴含
- `A & B`、`A | B`：作为定义缩写自动展开
- `@x A`：全称量词，也兼容 `∀x A`
- `?x A`：存在量词，也兼容 `∃x A`，作为 `~@x ~A` 的定义缩写
- 支持紧贴写法 `∃xP(x)`；如果量词要作用到复合公式，请写括号，例如 `∃x(P(x)∨Q(x))`
- 支持 Markdown/LaTeX 数学输入，例如 `$\forall x (P(x) \to Q(x))$`
- `P(x)`、`R(x,y,z,w)`：任意元数谓词
- `P`：零元谓词或公式模式变量
- `A,B=>C` 或 `$A,B \vdash C$`：带前提的目标；也支持 `$\vdash C$`

暂不支持等词与函词。

## 内置公理与规则

1. `A -> (B -> A)`
2. `(A -> (B -> C)) -> ((A -> B) -> (A -> C))`
3. `(~B -> ~A) -> (A -> B)`
4. `@x A -> A[x := t]`
5. `@x (A -> B) -> (A -> @x B)`，其中 `x` 不在 `A` 中自由出现

推理规则：

- MP：由 `A` 与 `A -> B` 推出 `B`
- UG：由 `A` 推出 `@x A`，其中 `x` 不在任何当前未消除假设中自由出现

## Markdown 导出

定理库里的每条定理都有“导出”按钮。导出时会递归收集该定理依赖的所有已保存定理，按拓扑顺序先输出引理证明，再输出目标定理证明，并直接复制 Markdown 到剪贴板；如果某段证明引用了引理，会在证明开头注明引用了哪些引理。导出的公式使用 Markdown 数学格式，例如 `$\forall x P(x) \to Q(x)$`。

如果某个定理或引理是通过演绎子证明完成的，导出时会自动识别临时假设，并用 `\vdash` 写出“由演绎定理命题转化为：$前提1, 前提2 \vdash 结论$”。

## 刷新恢复

应用会分别保存定理库和当前工作区。定理库跨窗口共享；当前目标、演绎栈和已经添加的证明步骤只在当前窗口会话中保存。刷新窗口 A 会恢复窗口 A 的证明，刷新窗口 B 会恢复窗口 B 的证明，新打开的窗口保持空白。

## 跨设备同步

使用 `run-site.bat` 打开时，定理库会同时保存到仓库文件 `data/theorems.json`。用 GitHub Desktop 同步时：

1. 在网页中保存、删除或清空定理库。
2. GitHub Desktop 会看到 `predicate-proof-lab/data/theorems.json` 发生变化。
3. 在 GitHub Desktop 中提交并推送。
4. 另一台设备拉取后，用 `run-site.bat` 打开页面，定理库会自动从 `data/theorems.json` 合并加载。

如果直接打开 `index.html`，浏览器无法写仓库文件，定理库仍会保存在本机浏览器中。
